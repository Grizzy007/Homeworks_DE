# Як запустити ноутбук заняття 02

Цей гайд показує, як відкрити та виконати `lesson_02_pct.ipynb` —
notebook заняття «Python для Data Engineering».

Усі приклади курсу розраховані на **VS Code**, тож основний шлях — саме через нього.
Якщо VS Code, Python чи Git ще не налаштовані — спочатку пройдіть загальний
setup-гайд курсу, а тоді поверніться сюди.

---

## Що знадобиться

- **Python 3.11+**
- **uv** — менеджер пакетів і віртуальних середовищ ([інструкція зі встановлення](https://docs.astral.sh/uv/getting-started/installation/))
- **VS Code** з розширеннями **Python** і **Jupyter**
- Інтернет для першого запуску (ноутбук завантажує датасет NYC TLC Yellow Taxi, ~50 MB)

> Перевірити, чи встановлено `uv`:
> ```bash
> uv --version
> ```

---

## Крок 1. Створити середовище і встановити бібліотеки

З кореня цього курсу перейдіть у папку коду заняття та створіть віртуальне середовище:

```bash
cd lesson-02-python/code
uv venv
```

Активуйте середовище:

```bash
# macOS / Linux
source .venv/bin/activate

# Windows (PowerShell)
.venv\Scripts\Activate.ps1
```

Встановіть залежності ноутбука:

```bash
uv pip install \
  pandas polars pyarrow requests icecream \
  matplotlib seaborn geopandas \
  jupyterlab ipykernel
```

> `ipykernel` потрібен, щоб VS Code «бачив» це середовище як kernel,
> а `jupyterlab` — для запуску без VS Code (Крок 3, альтернатива).

---

## Крок 2. Запустити ноутбук у VS Code (основний шлях)

1. Відкрийте папку `lesson-02-python` у VS Code
   (`File → Open Folder...`).
2. Відкрийте файл `code/lesson_02_pct.ipynb`.
3. Угорі праворуч натисніть **Select Kernel** і виберіть
   інтерпретатор зі створеного середовища — `.venv` (Python 3.11+).
4. Виконуйте клітинки по черзі — `Shift + Enter` —
   або одразу всі через **Run All** угорі.

Перша клітинка з завантаженням даних може виконуватися довше:
вона тягне Parquet-файл з даними поїздок. Запуск **ідемпотентний** —
якщо файл уже є в `../../data/source/`, повторне завантаження не відбувається.

---

## Крок 3 (альтернатива). Запустити без VS Code, через браузер

Якщо не хочете користуватися VS Code, з активованим середовищем запустіть:

```bash
jupyter lab
```

У вкладці браузера, що відкриється, перейдіть до
`lesson_02_pct.ipynb` і виконуйте клітинки так само — `Shift + Enter`.

## Структура файлів

```
code/
├── lesson_02_pct.ipynb   ← ноутбук, який запускаємо (основний deliverable)
├── lesson_02_pct.py      ← той самий код у форматі jupytext (percent)
```

Запускайте саме `lesson_02_pct.ipynb`. Файли `.py` — для тих, хто хоче
читати код у звичайному редакторі або робити diff у Git.

## Якщо щось пішло не так

- **`ModuleNotFoundError`** — середовище не активоване або вибрано не той kernel.
  Перевірте, що активний `.venv` (Крок 1) і що у VS Code обрано саме його (Крок 2.3).
- **Помилка під час `uv pip install geopandas`** — оновіть `uv`
  (`uv self update`) і повторіть; інсталяція йде з готових wheels.
- **Датасет не завантажується** — перевірте інтернет; за потреби
  перезапустіть першу клітинку, вона безпечна для повторного виклику.
- **VS Code не бачить kernel** — перевстановіть `ipykernel`
  у середовищі та перезавантажте вікно (`Developer: Reload Window`).
